Standbeine des WWW

  • Einheitliches Schema für das Auffinden von Ressourcen (URL/Hyperlink)
  • Protokolle, um auf diese Ressourcen zuzugreifen
  • Hypertext, um einfach zwischen Ressourcen zu navigieren (z.B. HTML)

HTTP 1.1

Request-Header | Response-Header

beinhalten Steuerinformationen, die Browser und Webserver miteinander austauschen.

Request-Header

  • Cache-Control: Zwischenspeichern der Daten (für eine bestimmte Zeit) auf Proxy
  • Host: www.hof-university.de \r \n
  • Content-Negotiation: Request- | Accept-Header
    • charset / Zeichensatz (Unicode)
    • encoding / gewählte Codierung (ASCII)
    • compression / Möglichkeit, Daten zu komprimieren
    • language / Sprache (de)

Response-Header

  • Expires: gibt an, wann die Seite durch den Browser neu angefordert werden muss
  • Location: gibt an, wohin der Inhalt verschoben wurde
  • Accept-Ranges:
  • Allow: Server gibt dem Browser an, was er alles kann
  • Server: Server stellt sich vor
HTTP-Content (Body)
Daten, die vom Anwender geschickt oder betrachtet werden.
